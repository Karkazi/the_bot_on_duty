# Редактирование сообщений в MAX (maxapi)

Проверка показала: **в maxapi есть полноценная поддержка редактирования сообщений.**

## Где в maxapi

- **Тип события:** `MessageEdited` — событие о том, что сообщение было отредактировано.
- **Метод у сообщения:** у объекта сообщения есть метод **`edit()`**.
- **Метод у бота:** `bot.edit_message(message_id=..., text=..., attachments=...)`.

## Как пользоваться

### 1. Редактирование из объекта сообщения (удобно в callback)

В обработчике нажатия на кнопку (`MessageCallback`) у вас есть `event.message` — это сообщение с кнопкой. Его можно отредактировать:

```python
# event — MessageCallback из @dp.message_callback()
await event.message.edit(
    text="Новый текст сообщения",
    attachments=new_buttons_list,  # те же типы, что и для answer()
)
```

Параметры `edit()` (из `maxapi/types/message.py`):

- **text** (str, optional) — новый текст.
- **attachments** (list, optional) — новые вложения (например, клавиатура `ButtonsPayload`).
- **link** — связь с сообщением (ответ/пересыл).
- **notify** (bool) — отправлять ли уведомление (по умолчанию True).
- **parse_mode** — режим разметки (HTML и т.п.).

Если `attachments` не передать, при редактировании сохраняются текущие вложения сообщения.

### 2. Редактирование через бота по message_id

Если есть только `message_id` и экземпляр бота:

```python
await bot.edit_message(
    message_id=message_id,  # str, из message.body.mid
    text="Новый текст",
    attachments=keyboard_attachments,
)
```

## Что нужно для спиннеров/календаря в MAX

1. В callback-обработчике при нажатии «+»/«−» или выборе даты:
   - брать текущее состояние из сессии MAX (аналог FSM);
   - считать новое значение;
   - формировать новый текст и новую клавиатуру;
   - вызывать **`await event.message.edit(text=..., attachments=...)`** — сообщение обновится на месте, без новых сообщений в чате.

2. **message_id** в callback не нужен отдельно: он уже есть у `event.message`, и `edit()` вызывается у этого объекта.

3. Документация API MAX: https://dev.max.ru/docs-api/methods/PUT/messages

## Итог

Редактирование сообщений в maxapi **есть** и **удобно** вызывается через `event.message.edit()`. Спиннеры и календарь в MAX реализуемы: при каждом нажатии кнопки можно обновлять одно и то же сообщение через `edit()`.

## Удаление сообщений бота после ответа (реализовано)

Чтобы в чате MAX оставался только финальный ответ бота («Сбой заведён», «Продлен», «Остановлен», «Сообщение отправлено» и т.д.):

- **Callback (нажатие кнопки):** перед обработкой вызывается `event.message.delete()` — удаляется сообщение бота с кнопкой. Новый ответ отправляется через `_reply_max_callback()`, его `message_id` сохраняется в сессии.
- **Текст:** перед отправкой нового ответа удаляется предыдущее сообщение бота по сохранённому `last_bot_message_id` (через `_reply_and_track()`). После отправки новый `message_id` сохраняется.
- При завершении сценария (успех, отмена) вызывается `clear_session()`, в котором сбрасывается и `last_bot_message_id`, поэтому финальное сообщение не удаляется при следующем действии.
