# Автооповещения по регламентным работам (Confluence)

## Ежедневная сводка (10:00 и 18:00)

- Фоновая задача: `services/calendar_digest_service.py` → `run_calendar_digest_scheduler()`.
- Получатели: `MAX_CALENDAR_ADMIN_IDS`.
- В сообщении — работы, которые **пересекают текущие календарные сутки** (00:00–23:59, `BOT_TIMEZONE`).
- Повтор в тот же слот в тот же день не отправляется (флаг в `state.json` → `calendar_digest_sent`).
- Пустой календарь: «На DD.MM.YYYY работ в календаре нет.»

Переменные `.env`:

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `CALENDAR_DIGEST_ENABLED` | `1` | `0` — отключить сводку |
| `CALENDAR_DIGEST_TIMES` | `10:00,18:00` | Слоты через запятую |

## Запрос «Информировать?» в момент inform_at

- Опрос Confluence каждые ~10 с: `services/confluence_calendar_worker.py`.
- При `now >= inform_at` дежурному уходит текст + кнопки «Информировать» / «Не информировать».
- Строки с **пустой** колонкой «Оповещения» в Confluence не обрабатываются.

## Подтверждение «Информировать»

- Каналы **не** читаются из колонки «Информирование бизнеса».
- Всегда: пост в **MAX-канал** (`MAX_ALARM_CHANNEL_ID`) и **Петлокал** (`create_maintenance` с `send_to_max=True`, `publish_petlocal=True`).
- Обработчик: `conf_notify_*` в `adapters/max/handlers.py`.

## Кнопка «Календарь» в главном меню MAX

- Показывает тот же список, что автосводка в 10:00 и 18:00 (работы на текущие сутки по `BOT_TIMEZONE`).
- Справка — только команда `/help` (кнопки «Помощь» в меню нет).

## Ручное добавление работы в календарь через бота

- Сценарий `cal_notify_*` — выбор каналов остаётся у дежурного (отдельно от автооповещения из Confluence).
