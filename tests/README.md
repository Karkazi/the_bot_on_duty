# Описание тестов

## 📋 Обзор

Тесты разделены на два типа:
- **Unit тесты** (`tests/unit/`) - тестируют отдельные функции и компоненты
- **Integration тесты** (`tests/integration/`) - тестируют взаимодействие компонентов

---

## 🧪 Unit тесты

### `test_validation.py` - Валидация входных данных

**Что проверяет:**
- ✅ **Санитизация HTML** (`sanitize_html`)
  - Экранирование HTML тегов (`<script>`, `</script>`)
  - Экранирование кавычек (`"`)
  - Экранирование амперсанда (`&`)

- ✅ **Валидация длины текста** (`validate_text_length`)
  - Валидная длина текста
  - Превышение максимальной длины

- ✅ **Валидация заголовка** (`validate_title`)
  - Валидный заголовок
  - Пустой заголовок
  - Заголовок только с пробелами

- ✅ **Валидация описания** (`validate_description`)
  - Валидное описание
  - Пустое описание

- ✅ **Валидация текста сообщения** (`validate_message_text`)
  - Валидный текст
  - Пустой текст

- ✅ **Валидация формата даты** (`validate_datetime_format`)
  - Валидная дата в правильном формате
  - Неверный формат даты
  - Некорректная дата (32.13.2025 25:00)

**Всего тестов:** 15

### `test_helpers.py` - Утилиты

**Что проверяет:**
- ✅ **Парсинг длительности** (`parse_duration`)

  - Пустая строка (возвращает None)

### `test_bot_state.py` - Управление состоянием

**Что проверяет:**
- ✅ **Создание аварии**
  - Авария успешно создается
  - Данные сохраняются корректно

- ✅ **Получение аварий пользователя** (`get_user_active_alarms`)

## 🔗 Integration тесты

### `test_services.py` - Бизнес-логика сервисов

#### `TestAlarmService` - Сервис управления авариями

**Что проверяет:**
- ✅ **Создание и получение аварии**
  - Авария создается с правильными данными
  - Авария сохраняется в `bot_state`
  - Аварию можно получить по ID

- ✅ **Продление аварии** (`extend_alarm`)
  - Время исправления корректно увеличивается
  - Новое время сохраняется в состоянии

- ✅ **Закрытие аварии** (`close_alarm`)
  - Авария удаляется из активных
  - Возвращаются данные закрытой аварии

- ✅ **Обработка ошибок валидации**
  - Выбрасывается `ValidationError` при пустом описании

- ✅ **Обработка отсутствия аварии**
  - Выбрасывается `NotFoundError` при запросе несуществующей аварии

**Всего тестов:** 5

#### `TestMaintenanceService` - Сервис управления регламентными работами

**Что проверяет:**
- ✅ **Создание и получение работы**
  - Работа создается с правильными данными
  - Работа сохраняется в `bot_state`
  - Работу можно получить по ID

- ✅ **Продление работы** (`extend_maintenance`)
  - Время окончания корректно увеличивается
  - Новое время сохраняется в состоянии

- ✅ **Обработка ошибок валидации**
  - Выбрасывается `ValidationError` при пустом описании

**Всего тестов:** 3

## 🚀 Запуск тестов

### Все тесты
```bash
pytest
```

### Только unit тесты
```bash
pytest tests/unit/
```

### Только integration тесты
```bash
pytest tests/integration/
```

### Конкретный файл
```bash
pytest tests/unit/test_validation.py
```

### С покрытием кода
```bash
pytest --cov=the_bot_on_duty --cov-report=html
```

### С подробным выводом
```bash
pytest -v
```

## 🔧 Конфигурация

### `conftest.py`
- **Фикстуры:**
  - `event_loop` - создает event loop для async тестов
  - `bot_state` - создает экземпляр BotState без очереди
  - `mock_bot` - создает мок бота для тестов

### `pytest.ini`
- Настройка pytest
- `asyncio_mode = auto` - автоматическая поддержка async тестов
